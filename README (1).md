## 프로젝트 개요

미카엘 챗봇은 **성당·성지 이름을 입력받아** ① 인원·일정·예산·컨셉에 맞춘 순례 여행을 제안하고, ② 미사시간·위치·개방여부·사제명 같은 사실 정보를 안내하는 챗봇이다. 신자·비신자·국내외 이용자 모두를 대상으로 한다.

| 항목 | 내용 |
|------|------|
| **타겟 사용자** | 국내 천주교 성지순례에 관심 있는 신자·비신자, 국내외 방문객 |
| **업무 문제** | 미사시간·개방여부·숙박·식당 정보를 따로 검색해야 하는 불편 해소. 인원·일정·예산에 맞는 맞춤 순례 일정 제공 |
| **입력** | 자유 질문 + 카테고리 보조 선택 |
| **출력** | 표와 이모티콘을 활용한 간결한 일정표·안내문 |

## 핵심 개념 3줄 요약 (동료평가 대비)

1. LLM은 많은 데이터를 학습해서 다음에 올 말을 확률적으로 예측해 생성한다. 그래서 사실 여부와 상관없이 그럴듯한 답을 만들어낼 수 있고, 이것이 환각이다.
2. Few-shot 예시를 3개 넣는 이유는 챗봇에게 모범 답안을 미리 보여줘서, 사용자의 질문에 맥락에 맞는 답변을 하도록 방향을 잡아주기 위해서다.
3. 모델 비교할 때 '느낌'으로 고르면 안 되는 이유는 주관적이기 때문이고, 그러면 동료평가에서 선정 근거를 설명할 수 없기 때문이다.
